Project Documentation

DGSIN-2526-10 is a cloud information system for managing CO₂ emission records and integrating them with external yearly electricity generation data from Ember Energy.

Project summary

The application provides a REST API and an Angular frontend to query, filter, create, edit and delete CO₂ emission records by location and year.

The main data source is a structured dataset about CO₂ emissions. The project also integrates an external API from Ember Energy through a backend proxy, avoiding the exposure of the external API key in the frontend.

Main API endpoints

Method Endpoint Description
GET /api/v1 Returns a JSON index with the main API resources.
GET /api/v1/health Checks API status.
GET /api/v1/openapi Shows the OpenAPI documentation using Swagger UI.
GET /api/v1/carbon-emission-records Lists and filters CO₂ emission records.
POST /api/v1/carbon-emission-records Creates a new CO₂ emission record.
DELETE /api/v1/carbon-emission-records Deletes all CO₂ emission records.
GET /api/v1/carbon-emission-records/:id Gets an existing CO₂ emission record.
PUT /api/v1/carbon-emission-records/:id Updates an existing CO₂ emission record.
DELETE /api/v1/carbon-emission-records/:id Deletes an existing CO₂ emission record.
GET /api/v1/carbon-emission-records/locations Returns available local dataset locations.
GET /api/v1/carbon-emission-records/loadInitialData Loads the initial dataset only if the collection is empty.
GET /api/v1/carbon-emission-records/docs Redirects to the public Postman documentation portal.
GET /api/v1/integrations/supported-locations Returns locations compatible between Ember Energy and the local dataset.
GET /api/v1/integrations/renewable-electricity Returns integrated local emissions and external Ember Energy data.
